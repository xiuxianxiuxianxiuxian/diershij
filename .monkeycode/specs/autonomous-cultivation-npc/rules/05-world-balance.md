# ==========================================
# 世界平衡与演化规则 (World Balance & Evolution)
# ==========================================

## 1. 世界生态健康度评估

### 1.1 核心指标
| 指标 | 计算方式 | 健康阈值 |
|------|----------|----------|
| 实力基尼系数 | `gini(entity_powers)` | < 0.7 |
| 资源流通率 | `tx_volume_24h / total_resources` | > 0.01 |
| 宗门多样性 | `sect_count / entity_count` | > 0.05 |
| 业力分布标准差 | `stddev(entity_karma)` | < 500 |

### 1.2 平衡干预策略

| 异常状况 | 天道自动干预 |
|----------|--------------|
| 实力差距过大 (基尼>0.7) | 为弱者生成机缘、古传承 |
| 资源流通过低 (<0.01) | 资源刷新率 ×1.5 |
| 宗门垄断 (>95% 人口) | 散修机缘增加，宗门内乱概率提升 |
| 业力极端分化 (stddev>500) | 触发天道清洗事件 |

## 2. 势力平衡规则

### 2.1 势力实力评估
```
power = member_power × 0.4 + resource_control × 0.4 + influence × 0.2
```

- `member_power`: 成员总战力
- `resource_control`: 控制区域灵气价值总和
- `influence`: 盟友数量 - 仇敌数量

### 2.2 制衡机制
当某势力实力超过全服平均 3 倍时：
1. 为弱势势力生成上古传承机缘
2. 弱势势力区域资源刷新 ×1.5
3. 该势力内部叛变概率 +10%

## 3. 灵气潮汐规则

### 3.1 周期
- 完整周期：365 天
- 计算公式：`tide = sin(2π × day / 365)`

### 3.2 潮汐阶段
| 阶段 | 阈值 | 灵气修正 | 持续时间 |
|------|------|----------|----------|
| 涨潮 | > 0.5 | +30% | 约 60 天 |
| 平稳 | -0.5 ~ 0.5 | 无 | 约 245 天 |
| 退潮 | < -0.5 | -30% | 约 60 天 |

### 3.3 特殊潮汐事件
- 百年大潮：每 100 年一次，灵气 +100%，突破成功率 +20%
- 灵气枯竭：天道惩罚，灵气 -80%，持续 30 天

## 4. 纪元更替规则

### 4.1 更替条件 (全部满足)
1. 元婴及以上实体数量 ≥ 10
2. 世界运行时间 ≥ 1000 年
3. 百年内发生重大冲突 ≥ 5 次

### 4.2 新纪元生成
1. **灵气重置**: 重新分布灵气浓度
2. **资源重分布**: 稀有资源刷新位置变更
3. **天道规则微调**: 部分算法参数随机 ±10%
4. **新秘境生成**: 2-3 个新秘境出现
5. **旧遗迹掩埋**: 部分旧区域永久封闭

## 5. 妖兽生成规则

### 5.1 生成率
```
rate = danger_level × 10 × spiritual_factor × human_suppression × tide_factor
```

- `spiritual_factor = density / 100`
- `human_suppression = 1.0 - (population / capacity) × 0.5`
- `tide_factor`: 妖兽潮期间最高 ×3.0

### 5.2 等级分布
基于区域危险等级的指数衰减分布：
```
P(level) = exp(-0.5 × (level - base_level))
```

### 5.3 妖兽潮周期
- 基础周期：每 3-5 年一次
- 业力过高地区频率翻倍
- 妖兽潮期间妖兽生成率 ×3，等级 +1

## 6. 资源刷新规则

### 6.1 刷新率
```
rate = base_rate × spiritual × pressure × balance
```

- `pressure = max(0.1, 1.0 - recent_harvest / base_rate × 10)`
  - 24 小时内采集越多，刷新越慢

### 6.2 稀有资源概率
```
P = 0.001 × tier_factor × exploration_factor
```

- `tier_factor = spiritual_tier / 9`
- `exploration_factor`: 区域探索活跃度
