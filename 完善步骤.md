## 工作流 E：AI NPC 系统（差异化核心）

### E1. LLM 集成（替代当前随机 stub）
- **文件**: `server/ai-scheduler/internal/service/ai_scheduler.go`
- **改动**: 实现真实的 DeepSeek API 调用，替代 `callLLM` 的随机选择
- **逻辑**: 构建 NPC 状态 prompt（境界/目标/性格/周围环境），调用 LLM 获取行为决策
- **参考**: 已有 `NPCProfile` 结构体（personality、ambition、risk_tolerance、background_story）

### E2. NPC 自主行为循环
- **文件**: `server/ai-scheduler/cmd/main.go`
- **改动**: 后台 goroutine 定时 tick，驱动所有 NPC 执行决策
- **逻辑**: 每 30-60 秒为每个活跃 NPC 调用 LLM → 执行行为树 → 提交操作到 game-server

### E3. NPC 与游戏世界交互
- **文件**: `server/ai-scheduler/internal/service/ai_scheduler.go`
- **改动**: NPC 决策结果转换为实际 game-server 操作（cultivate/explore/combat/trade）
- **逻辑**: 通过 gRPC 调用 game-server 的 `ExecuteOperation` 执行 NPC 行为

### E4. NPC 记忆与关系
- **文件**: `server/ai-scheduler/internal/service/`
- **改动**: 新建 `memory.go`，为 NPC 添加短期/长期记忆
- **逻辑**: NPC 记住与玩家的交互历史，影响后续决策（友好/敌对）

### E5. NPC 数据持久化
- **文件**: `server/ai-scheduler/internal/repository/`
- **改动**: 新建 NPC 持久化层，保存 NPC 状态到 PostgreSQL
- **逻辑**: 服务重启后 NPC 状态不丢失

## 工作流 K：Bubble Tea TUI 客户端

### K1. 项目搭建与框架集成
- **文件**: `cultivation-bubbletea/` 新建项目目录

- **底部固定**: 输入行 + 状态栏（境界/血量/位置）
- **参考**: `bubbles/viewport`、`bubbles/spinner`、`lipgloss` 样式

### K3. WebSocket 客户端包装
- **文件**: `cultivation-bubbletea/internal/client/ws.go`

### K4. 角色信息面板
- **文件**: `cultivation-bubbletea/internal/ui/character.go`

- **逻辑**: 灵力/修为用进度条展示（`bubbles/progress`）；灵根根据元素显示不同颜色

### K6. 战斗与日志面板

## 执行顺序建议

```
七开方案:
  实例1 → E (AI NPC)            — 核心差异化功能
  实例2 → F (商店/交易)         — 经济系统基础
  实例3 → G (世界事件)          — 世界动态感
  实例4 → H (社交完善)          — 社区粘性
  实例5 → I (宗门深化)          — 社交组织扩展
  实例6 → J (任务/成就)         — 目标导向玩法
  实例7 → K (BubbleTea TUI)     — 沉浸式终端体验
```

**推荐优先实现 E + F + G + K**，K 可以在任意阶段并行开发，与后端工作流无依赖。
